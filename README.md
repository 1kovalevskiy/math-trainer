# math-trainer

`math-trainer` - терминальный тренажер устного счета для детей. Он запускается как TUI-приложение: все управление происходит внутри терминала с клавиатуры или мышью.

Тренировка рассчитана на одну сессию. Пока приложение открыто, оно помнит текущие примеры и результаты. После выхода состояние тренировки очищается.


`Тренажер навайбкоден за пару десятков вечеров по предложению и идее доктора М.Д.`

## Главное меню

На стартовом экране доступны три пункта:

- `Начать тренировку` - сразу запускает сессию с текущими настройками.
- `Настройки сложности` - открывает выбор уровней и количества примеров.
- `Выход` - закрывает приложение.

Управление в меню:

- `↑`/`↓` или `k`/`j` - выбрать пункт.
- `Enter` - открыть выбранный пункт.
- `Click` - выбрать пункт мышью.
- `Ctrl+C` - закрыть приложение из любого экрана.

## Настройки тренировки

Перед стартом можно отдельно настроить:

- сложение;
- вычитание;
- умножение;
- деление;
- количество примеров в сессии.

Для каждой операции выбирается уровень сложности:

- `Отключено` - операция не попадает в тренировку.
- `Начальный`
- `Легкий`
- `Средний`
- `Сложный`
- `Экспертный`

Если отключить все операции, приложение автоматически вернет сложение на начальный уровень, чтобы тренировку можно было запустить.

Количество примеров ограничено диапазоном от `1` до `50`. По умолчанию используется `10` примеров.

Управление на экране настроек:

- `↑`/`↓` или `k`/`j` - переход между строками.
- `←`/`→` или `h`/`l` - изменить выбранное значение или выбрать кнопку.
- `Enter` - применить настройки или нажать выбранную кнопку.
- `Esc` - вернуться назад.
- `Click` - изменить значение или нажать кнопку мышью.

## Прохождение тренировки

Во время тренировки приложение показывает:

- номер текущего примера;
- уровень сложности для выбранной операции;
- сам пример;
- поле ответа;
- кнопки `Ответить`, `Пропустить`, `В меню`.

Чтобы решить пример, введите ответ цифрами и нажмите `Enter`, когда выбрана кнопка `Ответить`. Пример можно пропустить клавишей `S` или кнопкой `Пропустить`.

Управление на экране задания:

- цифры - ввод ответа;
- `Backspace` - удалить последнюю цифру;
- `←`/`→` или `h`/`l` - выбрать кнопку;
- `Enter` - нажать выбранную кнопку;
- `S` - пропустить пример;
- `Esc` - выйти в меню;
- `Click` - нажать кнопку мышью.

Примеры в одной тренировке не повторяются: приложение сравнивает уже решенные и пропущенные задания по левой части, правой части и операции.

## Результаты

После последнего примера открывается экран результатов. На нем видны:

- итоговый счет;
- длительность тренировки;
- использованные настройки сложности;
- список примеров с правильными, неправильными и пропущенными ответами.

Дальше можно:

- решить еще один пример с теми же настройками;
- перейти в настройки сложности;
- вернуться в главное меню.

Управление на экране результатов:

- `↑`/`↓`, `k`/`j` или колесо мыши - прокрутка списка;
- `PgUp`/`PgDn`, `Home`/`End` - быстрая прокрутка;
- `←`/`→` или `h`/`l` - выбрать кнопку;
- `Enter` - нажать выбранную кнопку;
- `Click` - нажать кнопку мышью.

## Сетка сложности

Операции включаются в тренировку независимо друг от друга. Если включено несколько операций, следующий пример выбирается случайно среди включенных операций.

### Сложение

| Уровень | Левая часть | Правая часть |
| --- | ---: | ---: |
| Начальный | 1-10 | 1-10 |
| Легкий | 10-99 | 10-99 |
| Средний | 50-999 | 50-999 |
| Сложный | 100-9999 | 100-9999 |
| Экспертный | 1000-9999 | 1000-9999 |

### Вычитание

| Уровень | Левая часть | Правая часть | Отрицательный результат |
| --- | ---: | ---: | --- |
| Начальный | 1-10 | 1-10 | Нет |
| Легкий | 10-99 | 10-99 | Нет |
| Средний | 50-999 | 50-999 | Нет |
| Сложный | 100-9999 | 100-9999 | Да |
| Экспертный | 1000-9999 | 1000-9999 | Да |

На уровнях без отрицательного результата приложение переставляет числа местами, если правая часть оказалась больше левой.

### Умножение

| Уровень | Левая часть | Правая часть |
| --- | ---: | ---: |
| Начальный | 1-10 | 1-10 |
| Легкий | 2-12 | 2-12 |
| Средний | 10-50 | 2-12 |
| Сложный | 10-99 | 10-20 |
| Экспертный | 50-100 | 50-100 |

### Деление

В делении приложение генерирует делитель и частное, а затем перемножает их для левой части примера. Поэтому все примеры делятся нацело.

| Уровень | Делитель | Частное | Пример выглядит как |
| --- | ---: | ---: | --- |
| Начальный | 1-10 | 1-10 | `(делитель * частное) / делитель` |
| Легкий | 2-20 | 2-20 | `(делитель * частное) / делитель` |
| Средний | 10-50 | 10-50 | `(делитель * частное) / делитель` |
| Сложный | 20-75 | 20-75 | `(делитель * частное) / делитель` |
| Экспертный | 50-100 | 50-100 | `(делитель * частное) / делитель` |

## Конфигурация

Файл конфигурации хранит последние примененные настройки тренировки:

```yaml
app:
  log_level: INFO
training:
  add_difficulty: starter
  subtract_difficulty: starter
  multiply_difficulty: disabled
  divide_difficulty: disabled
  examples_count: 10
```

Поддерживаемые значения сложности в YAML:

- `disabled`
- `starter`
- `easy`
- `medium`
- `hard`
- `expert`

Переменные окружения:

- `APP_CONFIG_PATH` - путь к YAML-конфигу.
- `APP_LOG_LEVEL` - уровень логирования: `DEBUG`, `INFO`, `WARN`, `WARNING`, `ERROR`.

## Для разработчика

Проект написан на Go `1.25` и использует Bubble Tea, Lip Gloss, bubblezone, cleanenv, godotenv и `slog`.

### Запуск

```bash
go run ./cmd
```

При первом запуске приложение создает YAML-конфиг `math-trainer.yaml` рядом с исполняемым файлом. Другой путь можно указать через переменную окружения:

```bash
APP_CONFIG_PATH=/path/to/math-trainer.yaml math-trainer
```

### Тестирование

Базовая проверка:

```bash
go test ./...
```

Если стандартный Go cache недоступен в sandbox-окружении:

```bash
GOCACHE=/private/tmp/math-trainer-go-cache go test ./...
```

Форматирование измененных Go-файлов:

```bash
gofmt -w <files>
```
